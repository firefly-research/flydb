    fsql [flags]
    fsql -e "<command>"
  Flags
    -H, --host <host>      Server hostname(s), comma-separated for HA cluster
    -p, --port <port>      Server port number (default: 8889)
    -d, --database <name>  Database to connect to (default: 'default')
    -v, --version          Print version information and exit
        --help             Show this help message
        --verbose          Enable verbose output with timing
        --debug            Enable debug mode with detailed logging
    -f, --format <format>  Output format: table, json, plain
        --no-color         Disable colored output
    -e, --execute <cmd>    Execute a command and exit
    -c, --config <file>    Path to configuration file
        --target-primary    Prefer connecting to primary in cluster
        --no-tls             Disable TLS and use plain TCP connection
        --tls-insecure        Skip TLS certificate verification (insecure)
        --tls-ca <file>      Path to CA certificate file for TLS verification
        --tls-cert <file>    Path to client certificate file for mutual TLS
        --tls-key <file>     Path to client key file for mutual TLS
  Examples
    # Connect to local server (uses 'default' database)
    fsql
    # Connect to a specific database
    fsql -d mydb
    # Connect to remote server
    fsql -H 192.168.1.100 -p 8889
    # Connect to HA cluster (multiple hosts)
    fsql -H node1,node2,node3 -p 8889
    # Execute a query and exit
    fsql -e "SELECT * FROM users"
    # Connect with TLS (enabled by default)
    fsql -H example.com
    # Connect with custom CA certificate
    fsql -H example.com --tls-ca /path/to/ca.crt
    # Connect with mutual TLS (client certificate and key)
    fsql -H example.com --tls-cert client.crt --tls-key client.key
    # Connect skipping TLS verification (for self-signed certs)
    fsql -H example.com --tls-insecure
  Interactive Commands
    \q, \quit         Exit the CLI
    \h, \help         Display help information
    \c <db>          Switch to database
    \clear            Clear the screen
    \s, \status       Show connection status
  Environment Variables
    FLYDB_HOST        Default server host
    FLYDB_PORT        Default server port
    FLYDB_DATABASE    Default database name
    NO_COLOR          Disable colored output
  For more information, visit: https://github.com/firefly-oss/flydb

USAGE:
  flydb [options]
  flydb                    # Start with interactive wizard
CORE OPTIONS:
  -port <port>      Server port for binary protocol connections (default: 8889)
  -role <role>      Server role: standalone, cluster (default: standalone)
  -data-dir <path>     Directory for database storage (default: /Users/ancongui/.local/share/flydb)
  -config <file>     Path to configuration file
  -version             Show version information and exit
  -help                Show this help message
SECURITY OPTIONS:
  -tls-enabled      Enable TLS encryption (default: true)
  -tls-auto-gen         Enable auto-generation of TLS certificates (default: true)
  -tls-cert-file <file>    Path to TLS certificate file
  -tls-key-file <file>     Path to TLS private key file
  -encryption-enabled   Enable data-at-rest encryption (default: true)
CLUSTER OPTIONS:
  -cluster-port <port>   Cluster communication port (default: 9998)
  -repl-port <port>      Replication port for cluster sync (default: 9999)
  -cluster-peers <peers> Comma-separated list of cluster peers (host:port)
  -enable-raft         Enable Raft consensus for leader election (default: true)
PERFORMANCE OPTIONS (01.26.17+):
  -enable-compression  Enable compression for WAL and replication (default: false)
  -compression-algorithm <algo>  Compression algorithm: lz4, snappy, zstd (default: lz4)
  -enable-zero-copy   Enable zero-copy buffer pooling (default: true)
  -buffer-pool-size-bytes <n> Buffer pool size in bytes, 0 = auto (default: 0)
OBSERVABILITY & LOGGING:
  -log-level <level>   Log level: debug, info, warn, error (default: info)
  -log-json            Enable JSON log output format
  -audit-enabled       Enable comprehensive audit logging (default: true)
  -audit-retention-days <n> Days to retain audit logs (default: 90)
ENVIRONMENT VARIABLES:
  FLYDB_DATA_DIR         Data directory for database storage
  FLYDB_PORT             Server port for client connections
  FLYDB_ENCRYPTION_PASSPHRASE  Encryption passphrase (required if encryption enabled)
  FLYDB_ADMIN_PASSWORD       Admin password for first-time setup
EXAMPLES:
  # Start with interactive wizard
  flydb
  # Start standalone server with encryption
  export FLYDB_ENCRYPTION_PASSPHRASE="secure-passphrase"
  flydb -role standalone -data-dir ./data
  # Start cluster node
  flydb -role cluster -cluster-peers node1:9998,node2:9998
CONNECTING:
  Use fsql to connect to the server:
    fsql -H localhost -p 8889
FlyDB Dump Utility v1.0.0 - Database export and import tool
────────────────────────────────────────────────────────────
USAGE:
  Local mode (direct file access):
    fdump -d <data_dir> [options]
  Remote mode (network connection):
    fdump --host <hostname> [--port <port>] [options]
CONNECTION OPTIONS:
  -d <path>           Data directory path (local mode)
  --host <hostname>   Remote server hostname(s), comma-separated for cluster
  --port <port>       Remote server port (default: 8889)
  -db <name>          Database name (default: default)
  --connect-timeout <dur> Connection timeout (default: 10s)
  --query-timeout <dur> Query timeout (default: 30s)
EXPORT OPTIONS:
  -o <file>           Output file path (default: stdout)
  -f <format>         Output format: sql, csv, json (default: sql)
  -t <tables>         Comma-separated list of tables (default: all)
  --schema-only       Dump schema only, no data
  --data-only         Dump data only, no schema
  --no-owner          Do not output ownership commands
  -z                  Compress output with gzip
IMPORT OPTIONS:
  --import <file>     Import data from SQL dump file
AUTHENTICATION:
  -U <username>       Username for authentication
  -W <password>       Password (use -P for secure prompt)
  -P                  Prompt for password interactively
TLS OPTIONS (remote mode):
  --no-tls            Disable TLS (use plain TCP)
  --tls-insecure      Skip TLS certificate verification (insecure)
  --tls-ca <file>     Path to CA certificate file for TLS verification
  --tls-cert <file>   Path to client certificate file for mutual TLS
  --tls-key <file>    Path to client key file for mutual TLS
ENCRYPTION (local mode only):
  --passphrase <pass> Encryption passphrase
  --prompt-passphrase Prompt for passphrase interactively
OTHER OPTIONS:
  -v                  Verbose output
  --version           Show version information
  --help              Show this help message
EXAMPLES:
  # Local mode - Full SQL dump
  fdump -d ./data -o backup.sql
  # Local mode - Dump with authentication
  fdump -d ./data -U admin -P -o backup.sql
  # Local mode - Dump encrypted database
  fdump -d ./data --passphrase secret -o backup.sql
  # Remote mode - Dump from server
  fdump --host localhost --port 8889 -U admin -P -o backup.sql
  # Remote mode - Cluster dump (connects to any node)
  fdump --host node1.example.com,node2.example.com -U admin -P -o backup.sql
  # Remote mode - Import to cluster (discovers leader)
  fdump --host node1.example.com -U admin -P --import backup.sql
  # Export specific tables as JSON
  fdump --host localhost -t users,orders -f json -o data.json
  # Schema only dump
  fdump --host localhost --schema-only -o schema.sql
  # Compressed dump
  fdump --host localhost -z -o backup.sql.gz
  # Local mode - Import from dump file
  fdump -d ./data --import backup.sql
ENVIRONMENT VARIABLES:
  FLYDB_USER                  Default username for authentication
  FLYDB_ADMIN_PASSWORD        Password for authentication
  FLYDB_ENCRYPTION_PASSPHRASE Encryption passphrase (local mode)
  For more information, visit: https://github.com/firefly-oss/flydb

  FlyDB Discover v01.26.17
  Network Node Discovery Tool
  Discovers FlyDB nodes on the local network using mDNS (Bonjour/Avahi).
  Useful for finding existing cluster nodes to join.
  USAGE:
    flydb-discover [options]
  OPTIONS:
    --timeout <n>     Discovery timeout in seconds (default: 5)
    --json            Output results as JSON
    --quiet, -q       Only output addresses (for scripting)
    --version, -v     Show version information
    --help, -h        Show this help message
  EXAMPLES:
    # Discover nodes with default timeout
    flydb-discover
    # Get JSON output for automation
    flydb-discover --json
  NETWORK REQUIREMENTS:
    • mDNS uses UDP port 5353 (multicast)
    • Nodes must be on the same network segment
